NABU PC MAME - All-In-One (AIO) AppImage Build for Steam Deck & Linux
======================================================================

Version 1.03 - NABUPcAIO

The all-in-one AppImage bundles NABU PC MAME with all three supported
servers in a single executable. Choose your server at launch with the
--server flag — no separate downloads or installs required.

By default (no options) MAME connects directly to nabu.ca using its
built-in hcca_remote connection — no bundled server needed.


Bundled servers:

  NIA   NABU Internet Adapter (DJ Sures) — connects to nabu.ca retronet
        Runs headless in the background.

  NNS   NABU NetSim v1.3.4-release5 (Nick Daniels / GryBsh) — local
        advanced NABU simulator with web UI at http://localhost:5000

  FujiNet  CoCo FujiNet v1.5.1 (FujiNet Team) — 8-bit networking for
        CoCo3 via MAME. Web UI at http://localhost:8000

This is a Linux 64-bit build targeted at the Steam Deck. It should run
on most modern Linux distros. Built and tested on Linux Mint and WSL2.


Data file locations:

  ~/.mamedata/            MAME configuration and ROM files
  ~/Pictures/nabupc/      MAME screen captures
  ~/Documents/NABU Internet Adapter/   NIA binary and config
  ~/.nns/                 NNS runtime data (created on first run)
  ~/Documents/FujiNetCoCo/             FujiNet binary and config

All data folders are created automatically on first run and restored to
defaults with --reset (existing configs backed up to .old).

NIA can be upgraded by replacing the binary in
~/Documents/NABU Internet Adapter/ with a newer version.
NNS is a fixed bundled version. FujiNet can be updated by replacing
the contents of ~/Documents/FujiNetCoCo/.


Basic syntax:

  ./NABUPcAIO.AppImage [options] [mame options]


Options:

  --help              Display full help and exit
  --server <name>     Server to run: none (default), nia, nns, fuji
  --fuji              Shorthand for --server fuji
  --nonia             Shorthand for --server none (no server)
  --nonns             Shorthand for --server none (no server)
  --mame              MAME only — pass your own command directly
  --coleco            Run built-in Coleco ROM collection (brijohn)
  --config            Open server config UI:
                        NIA     — GUI window, exits cleanly when closed
                        NNS     — prints http://localhost:5000, press Enter to stop
                        FujiNet — prints http://localhost:8000, press Enter to stop
  --reset             Reset all configs to defaults and exit
  --debug             Enable debug output
  --about             Display this file


Examples:

  ./NABUPcAIO.AppImage
    MAME connects directly to nabu.ca via hcca_remote (default).

  ./NABUPcAIO.AppImage --server nia
    Launch NIA and MAME — nabu.ca session via bundled NIA server.

  ./NABUPcAIO.AppImage --server nia --config
    Open NIA config GUI — exits cleanly when you close the window.

  ./NABUPcAIO.AppImage --server nns
    Launch NNS and MAME — local simulator session.

  ./NABUPcAIO.AppImage --server nns --config
    Start NNS and print the web UI URL — press Enter to stop server.

  ./NABUPcAIO.AppImage --server fuji
    Launch CoCo FujiNet and MAME with CoCo3.

  ./NABUPcAIO.AppImage --coleco
    Run the built-in Coleco ROM collection — no server needed.

  ./NABUPcAIO.AppImage --mame nabupc -noswitchres -resolution 1024x768 -nabu_video tms9938 -hcca null_modem -bitb socket.192.168.0.10:5816
    MAME only with a fully custom command, connecting to a local server.

All standard MAME command line options are passed directly to MAME.


Bonus Cores — Vectrex and CoCo3
=================================

The bundled MAME binary includes Vectrex and CoCo3 / CoCo3h emulation
in addition to NABU PC. Place your own ROM files in
~/.mamedata/roms/<system>/ before use.

Use --mame for all bonus core sessions — this bypasses all servers and
passes your command directly to MAME.


Vectrex
-------

Basic launch (BIOS screen / built-in Mine Storm game):
  ./NABUPcAIO.AppImage --mame vectrex -window -rompath ~/.mamedata/roms

With a cartridge:
  ./NABUPcAIO.AppImage --mame vectrex -window -rompath ~/.mamedata/roms -cart ~/.mamedata/roms/vectrex/scramble.zip

Debug mode to confirm what is being passed to MAME:
  ./NABUPcAIO.AppImage --debug --mame vectrex -window -rompath ~/.mamedata/roms


CoCo3 and CoCo3h
-----------------

Two CoCo3 variants are available:
  coco3   — Motorola 6809 CPU (original hardware)
  coco3h  — Hitachi HD6309 CPU (faster; use for HD6309 software)

Basic launch (BASIC prompt, no disk):
  ./NABUPcAIO.AppImage --mame coco3 -window -rompath ~/.mamedata/roms

With a floppy disk image (.DSK):
  ./NABUPcAIO.AppImage --mame coco3h -window -rompath ~/.mamedata/roms -ramsize 512k -flop1 ~/Documents/coco/GUNSTAR.DSK

With multiple disk images:
  ./NABUPcAIO.AppImage --mame coco3h -window -rompath ~/.mamedata/roms -ramsize 512k -flop1 ~/Documents/coco/DISK1.DSK -flop2 ~/Documents/coco/DISK2.DSK

Note: For CoCo FujiNet (network/SD card), use --server fuji instead.


This is an AppImage — a self-contained 64-bit Linux x86 executable.
All required libraries are included.
